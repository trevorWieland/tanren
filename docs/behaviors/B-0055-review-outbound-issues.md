## Intent

A `team-dev` with the required permission can configure the project to
queue outbound issues for human review instead of pushing them
automatically, and can review and approve or discard queued issues, so
that a team can avoid noisy or inappropriate filings in their external
tracker.

## Preconditions

- The project has a connected external tracker (B-0052).
- For setting the review mode: the user has permission to change
  project-tier configuration (B-0049) for the project.
- For approving or discarding queued items: the user holds the permission
  to approve outbound issues for the project, granted per the project
  access model (B-0031) or via a role template (B-0038).

## Observable outcomes

- The user can set the project's outbound-issue mode to either "push
  automatically" or "queue for review."
- When in review mode, issues generated by loops are held in a queue and do
  not appear in the external tracker until approved.
- A user with permission to approve outbound issues can review each queued
  item, edit its title or description before approval, approve it to push,
  or discard it.
- The default mode is chosen to match the team's preference; nothing forces
  review on projects that do not want it.

## Out of scope

- Multi-person approval workflows for a single outbound issue.
- Routing review of different issue categories to different people.
