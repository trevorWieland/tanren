## Intent

A `solo-builder` or `team-builder` can capture the product mission for a project so that future roadmap and spec work has a stable north star.

## Preconditions

- An active project is selected.
- The user has permission to edit product planning context.

## Observable outcomes

- The project records the product problem, target users, solution frame, and success criteria.
- The mission is visible to later planning and shaping work.
- Open mission decisions are called out rather than silently omitted.

## Out of scope

- Generating implementation tasks directly.
- Replacing user approval of product direction.
