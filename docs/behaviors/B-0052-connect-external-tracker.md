## Intent

A `solo-dev` or `team-dev` with the required permission can connect a
project to an external issue tracker such as Linear, Jira, or GitHub
Issues, so that the project can receive outbound issues generated by
implementation loops and reference external tickets as the origin of
specs.

## Preconditions

- The user has permission to change project-tier configuration for the
  project (see B-0049).
- The user has credentials or authorization sufficient for the project to
  push to and read from the chosen tracker.

## Observable outcomes

- The user can select a supported tracker and connect the project to a
  specific destination within it (for example, a Linear team or a GitHub
  repository's issues).
- After connection, outbound issues generated by the project's loops are
  routed to the connected destination (see B-0054).
- The user can change or disconnect the tracker later. Disconnecting does
  not retract issues that have already been pushed.
- A project can have at most one connected external tracker at a time.

## Out of scope

- Building or configuring the external tracker itself.
- Two-way state sync between a Tanren spec and an external ticket — the
  relationship remains one-way per concepts.md.
