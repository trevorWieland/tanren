# Evidence Document Schemas

Evidence artifacts are deterministic projections from
`phase-events.jsonl`. Markdown artifacts use **typed YAML frontmatter +
deterministic rendered body**. JSON/JSONL artifacts are versioned and
strictly validated. This doc is the authoritative schema contract.

Companion docs:
[orchestration-flow.md](orchestration-flow.md),
[agent-tool-surface.md](agent-tool-surface.md).

derive `Serialize`, `Deserialize`, and where appropriate `JsonSchema`
(from `schemars`). What follows is the canonical shape.

## 3. Validation lifecycle

### 3.1 On tool call

Every mutating methodology tool:
1. Validates the input against its serde schema.
2. Appends one typed event to the canonical store + outbox.
3. Projects `phase-events.jsonl` line envelopes (strict schema checks).
4. Folds all events for the spec and atomically rewrites the generated
   artifact set (`spec.md`, `plan.md`, `tasks.md`, `tasks.json`,
   `demo.md`, `audit.md`, `signposts.md`, `progress.json`,
   `phase-events.jsonl`, manifest).

Invalid input ŌåÆ `ToolError::ValidationFailed`; no file changes.

### 3.2 On agent session end

Postflight scans each evidence file:
- Re-parse frontmatter via serde; schema failure ŌåÆ
  `EvidenceSchemaError { file, error, line?, field? }`.
- Diff current body against pre-phase snapshot; non-body edits to
  frontmatter ŌåÆ `UnauthorizedArtifactEdit`; revert.
- Extract narrative body section headers; unknown or missing
  recommended headers ŌåÆ `DocumentationWarning { file, expected }`
  (non-fatal; surfaced in audit rubric under `documentation_complete`).

### 3.3 Error flow

- Validation error during agent session ŌåÆ typed error returned; agent
  may recover or exit.
- Schema error in postflight ŌåÆ phase outcome `Error`; orchestrator
  routes to `investigate`.
- Invalid event on replay ŌåÆ replay fails atomically; no partial
  store mutation.

## 4. Round-trip guarantees

Property-tested in methodology domain/app services:

1. `parse(render(frontmatter)) == frontmatter` for every type.
2. `render` produces deterministic output (sorted keys for maps where
   order is not semantic).
3. `parse` tolerates trailing whitespace, BOM, and CRLF.
4. Generated projection output is deterministic from the same event
   stream.
5. Unknown frontmatter keys are rejected (strict mode) ŌĆö prevents
   silent data loss.

## 5. Body conventions

### 5.1 Recognized section headers

Used for `DocumentationWarning` detection (non-fatal heuristics):

- `spec.md`: `## Motivation`, `## Background`, `## Scope`,
  `## Non-Negotiables`, `## Acceptance Criteria`, `## Demo Plan`,
  `## Task Breakdown`
- `demo.md`: `## Environment`, `## Steps`, `## Results`
- `audit.md`: `## Summary`, `## Rubric`, `## Non-Negotiables`,
  `## Findings`, `## Recommendations`
- `signposts.md`: per-entry sub-headers rendered deterministically
  from frontmatter; free narrative allowed at file bottom.

### 5.2 Renderer guidance

Generated narrative should be terse and specific. Structured facts
belong in typed fields/events and are never hand-edited in
orchestrator-owned artifacts.
